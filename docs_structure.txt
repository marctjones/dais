## Documentation Strategy

**During Alpha/Beta:** All docs in repository
- Easier to maintain during rapid changes
- Stays in sync with code versions
- Part of PR review process

**After v1.0:** Consider enabling wiki for:
- User guides and tutorials
- Community troubleshooting
- Instance showcase
- FAQ

## Style Guide

- Use GitHub-flavored Markdown
- Include code examples with syntax highlighting
- Add table of contents for long docs
- Keep line length reasonable (80-100 chars)
- Use emojis sparingly for section markers
